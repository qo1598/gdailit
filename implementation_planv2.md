# 16개 미션 기반 AI 리터러시 전문가 타당도 검토(세부 명세서) - 최종본 (KOR)

본 계획서는 전문가(AI 교육 학자, 현장 교사, 측정 평가자)가 16개 미션을 꼼꼼하게 검증할 수 있도록, **각 미션이 1) 어떤 역량(K-S-A)과 연결되어 있고, 2) 무엇을 측정하며, 3) 학생은 어떻게 수행하고, 4) 어떤 형태(JSON)의 연구 데이터가 수집되는지**를 한국어로 상세히 공개한 설계 명세서입니다.

---

## 🏛️ 영역 1: AI 알아보기 (Engaging)

### [E-1] 우리 주변의 AI 찾기
*   **연결 역량**
    *   **[K] 지식**: **K1.4**: 생성, 예측, 추천 등 목적에 따라 다르게 작동하는 AI 시스템의 특성 이해
    *   **[S] 기능**: **Self and Social Awareness**: AI가 개인의 선택과 관계 및 공동체에 어떻게 영향을 미치는지 인식
    *   **[A] 태도**: **Curious**: 오늘날의 AI 능력을 탐구하고 미래 발전을 탐색하고자 하는 호기심
*   **측정 목표**: 단순 전자기기와 AI 기기를 구분(식별)하고, 기술의 편의성 및 한계를 비판적으로 파악할 수 있는지 측정.
*   **플랫폼 수집 로그 (JSON)**: 
    ```json
    {
      "file_url": "https://...",
      "stackedAnswers": {
        "step1_location": "거실 로봇청소기",
        "step2_function": "먼지를 빨아들이고 바닥을 닦아요",
        "step3_analysis": "장점: 시간 절약, 단점: 구석 청소 불가"
      }
    }
    ```

### [E-2] AI 답 검증하기
*   **연결 역량**
    *   **[K]**: **K4.3**: 생성형 AI가 사실과 허구를 구별하기 어렵게 만들어 오정보를 생성할 수 있는 한계 인지
    *   **[S]**: **Critical Thinking**: 정확성, 공정성, 편향성을 고려하여 AI 생성 콘텐츠를 비판적으로 평가하는 기능
    *   **[A]**: **Responsible**: 자신의 선택과 AI 결과물에 대해 책무를 가지려는 책임감 있는 태도
*   **측정 목표**: AI의 환각 현상을 인지하고, 독자적으로 진위 여부를 판별하는 능력을 증명.
*   **플랫폼 수집 로그 (JSON)**: 실시간 논쟁 채팅 기록 (Chat History)
    ```json
    {
      "chat_history": [
        {"role":"ai", "content":"지구에서 두 번째로 큰 동물은 호랑이예요!"},
        {"role":"user", "content":"아니야, 백과사전에서 보니 긴수염고래야."}
      ]
    }
    ```

### [E-3] 추천은 왜 나에게 왔을까?
*   **연결 역량**
    *   **[K]**: **K1.1**: 데이터를 처리하고 패턴을 감지하여 확률적 결과를 생성하는 알고리즘 원리 이해
    *   **[S]**: **Self and Social Awareness**: AI 알고리즘이 개인의 추천과 기회에 미치는 영향을 인식하고 성찰함
*   **측정 목표**: 알고리즘과 필터 버블(Filter Bubble) 현상을 메타적으로 인지하는지 측정.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "rec_item": "슬라임 영상",
        "guess_reason": "어제 슬라임을 검색했기 때문이에요",
        "critical_reflection": "내가 좋아하는 것만 보게 되어 새로운 정보를 알기 어려워져요"
      }
    }
    ```

### [E-4] AI는 누구에게 불공평할까?
*   **연결 역량**
    *   **[K]**: **K2.5**: 학습 데이터나 알고리즘 설계에 내재된 사회적 편향(Bias)에 대한 이해
    *   **[S]**: **Critical Thinking**: AI 시스템의 공정성 및 편향성을 평가하고 윤리적 결정을 내리는 기능
    *   **[A]**: **Empathetic**: AI 기술이 개인이나 특정 공동체에 미치는 잠재적 위험과 영향을 사려 깊게 검토함
*   **측정 목표**: 데이터 편향성에 따른 차별적 출력 한계를 인지하고 윤리적 규범을 제안할 수 있는지 측정.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "bias_case": "카메라가 아이들의 작은 키를 인식하지 못했어요",
        "ethics_rules": "1. 다양한 데이터를 모은다. 2. 테스트를 여러 번 한다."
      }
    }
    ```

## 🎨 영역 2: AI와 함께 만들기 (Creating)

### [C-1] AI와 이야기 이어쓰기
*   **연결 역량**
    *   **[K]**: **K1.3**: 생성형 AI가 확률을 사용하여 인간과 유사한 결과물을 생성하는 원리 인식
    *   **[S]**: **Creativity**: AI와 협업하여 독창적인 아이디어를 생성하고 정교화하는 기능
    *   **[A]**: **Innovative**: 기술을 활용해 실생활의 과제에 창의적으로 도전하고 새로운 기회를 수용함
*   **측정 목표**: AI와 협업하되 주도권(Psychological Authorship)을 유지하며 기계 기여분을 인지하는지 확인.
*   **플랫폼 수집 로그 (JSON)**: 동화 창작 채팅 및 기여도 평가 기록
    ```json
    {
      "chat_history": [
        {"role":"ai", "content":"옛날 옛적 초콜릿 숲에..."},
        {"role":"user", "content":"결말은 내가 지었어. 내 기여도는 60%야."}
      ]
    }
    ```

### [C-2] AI 그림 보고 새 장면 만들기
*   **연결 역량**
    *   **[K]**: **K1.4**: 제작, 이미지 생성 등 목적에 따라 특화되어 작동하는 AI의 속성 이해
    *   **[S]**: **Creativity**: 소유권과 책임 있는 사용을 고려하며 AI 결과물을 역분석하거나 개선하는 기능
    *   **[A]**: **Adaptable**: AI와 협업할 때 유연성을 보이고 개선 과정에 끈기 있게 참여함
*   **측정 목표**: 이미지의 시각적 결함을 발견하고, 프롬프트를 역설계(Reverse Engineering)하는 능력 측정.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "selected_img": "마법 방패 그림",
        "prompt_reverse_engineering": "beautiful forest, realistic, 4k..."
      }
    }
    ```

### [C-3] AI와 홍보물 만들기
*   **연결 역량**
    *   **[K]**: **K5.3**: 기존 학습 데이터를 바탕으로 콘텐츠 생성 시 발생하는 저작권/출처 문제 파악
    *   **[S]**: **Creativity**: 소유권과 기여도(Attribution)를 고려하면서 AI와 협업하여 창작
    *   **[A]**: **Responsible**: AI 사용 방식이 타인에게 미칠 수 있는 영향에 대해 신중히 고려함
*   **측정 목표**: AI 출력물을 목적에 맞게 윤색(Refine)하며 표절/저작권 요소를 검토하는 윤리성 확인.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "refined_text": "우주 최강 경동초 5학년 합창단!",
        "integrity_check": "이미지 저작권을 확인했다."
      }
    }
    ```

### [C-4] 누가 만들었을까?
*   **연결 역량**
    *   **[K]**: **K5.3**: 저작물 보호와 생성물의 진위(Authenticity) 및 소유권 문제 이해
    *   **[S]**: **Communication**: 기술 사용 방식을 투명하게 설명하고 책임 있게 사용하는 소통 기능
    *   **[A]**: **Responsible**: 자신의 창작 선택에 책무를 가지며 정직하게 도구를 활용하는 태도
*   **측정 목표**: 저작성 딜레마를 이해하고, 생성물 활용 시 출처 표기 등 정직성 규범을 세우는 태도 확인.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "attribution_opinion": "AI에게 힌트를 얻었다고 솔직히 말한다.",
        "my_ethics_code": "나는 AI 결과물을 내가 온전히 만들었다고 속이지 않는다."
      }
    }
    ```

## ⚙️ 영역 3: AI 똑똑하게 사용하기 (Managing)

### [M-1] 이럴 때 AI 써도 될까?
*   **연결 역량**
    *   **[K]**: **K3.3**: 최종 결정과 윤리적 실천에 대한 책임은 인간에게 있음을 이해함
    *   **[S]**: **Problem Solving**: AI의 역량과 위험을 평가하여 과업 수행 시 사용 시점을 결정함
    *   **[A]**: **Responsible**: 자신의 선택이 가져올 의도치 않은 결과까지 신중하게 고려하는 태도
*   **측정 목표**: 단기적 편향과 장기적 학습 사이에서 인지적 노력을 유지해야 하는 이유(에티켓) 판단.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "etiquette_1": "정답만 베끼지 않는다",
        "etiquette_2": "모르는 개념의 힌트만 얻는다",
        "etiquette_3": "AI 정보를 무조건 믿지 않는다"
      }
    }
    ```

### [M-2] 사람이 할 일, AI가 도울 일
*   **연결 역량**
    *   **[K]**: **K3.2**: 어떤 과업이 기계에 가장 적합하고 어디에 인간의 전문성이 필요한지 결정함
    *   **[S]**: **Collaboration**: AI 및 인간과 효과적으로 협력하고 피드백을 주며 과업 공유
    *   **[A]**: **Adaptable**: 협력 결과물 완성을 위해 유연하게 대처하는 태도
*   **측정 목표**: 과업을 분해(Decompose)하고 휴먼-인-더-루프(HITL) 설계 능력을 발휘하는지 확인.
*   **플랫폼 수집 로그 (JSON)**: 인간과 AI의 역할 분담 협상 채팅
    ```json
    {
      "chat_history": [
        {"role":"ai", "content":"제가 어떤 것들을 도맡아 할까요?"},
        {"role":"user", "content":"너는 대본 수정을 돕고, 감정 연기 연습은 내가 할게!"}
      ]
    }
    ```

### [M-3] 정확한 지시
*   **연결 역량**
    *   **[S]**: **Computational Thinking**: 과제를 분해하고 AI가 효과적으로 기여하도록 정교한 지시(Prompt) 설계
    *   **[A]**: **Adaptable**: 피드백과 수정을 거쳐 반복적으로 결과물을 개선해나가는 성실함
*   **측정 목표**: 기계 언어 문법에 맞춰 오해 없는 지시(Prompt Formula)를 창조하는 능력 측정.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "prompt_formula": "[주제 + 분위기 + 색감]",
        "prompt_example": "강아지가 우주에 있는 모습을 쨍한 색감으로"
      }
    }
    ```

### [M-4] 우리 반 AI 사용 약속
*   **연결 역량**
    *   **[K]**: **K5.4**: 윤리적 AI 디자인(공정성, 투명성, 책임성 등)과 규범 체계에 대한 이해
    *   **[S]**: **Communication**: 투명성을 촉진하고 사이버 공간 내 책임 있는 기술 사용 권장
    *   **[A]**: **Responsible**: 기술이 타인에게 미치는 영향을 파악하고 행동에 책무를 다함
*   **측정 목표**: AI 활용 가이드라인을 공동체의 룰로 확장 내면화하는 디지털 시민성(Citizenship) 성취.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "proposals": "서로 정직하게 밝히기",
        "class_oath": "우리는 사람을 돕기 위해 AI를 씁니다."
      }
    }
    ```

## 🧪 영역 4: AI 원리 체험하기 (Designing)

### [D-1] 같은 것끼리 묶어보자
*   **연결 역량**
    *   **[K]**: **K1.2**: 기계가 데이터를 통해 학습하여 패턴과 분류 결과물을 생성하는 방식의 이해
    *   **[S]**: **Computational Thinking**: 문제를 구성요소로 구조화하고 논리적으로 분류하는 지시 기능
    *   **[A]**: **Curious**: 기술의 작동 과정과 원리를 깊이 탐구하고 이해하려는 지적 호기심
*   **측정 목표**: 머신 러닝의 분류 특징(Feature) 기준 추출 원리와 자의성을 인지하는지 평가.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "basic_cluster": "사과, 딸기",
        "custom_cluster": "씨가 큰 과일: 복숭아",
        "analysis": "분류 기준에 따라 AI의 판단이 달라지기 때문"
      }
    }
    ```

### [D-2] AI가 배우는 자료 모으기
*   **연결 역량**
    *   **[K]**: **K2.2**: 센서와 상호작용 등 பல்வேறு 소스로부터 학습 데이터셋이 구축됨을 파악
    *   **[S]**: **Problem Solving**: 데이터의 신뢰성과 위험성을 평가하여 과업에 적합하게 준비하는 기능
    *   **[A]**: **Responsible**: 데이터 편향이 초래할 위험성을 인식하고 신중히 대처함
*   **측정 목표**: GIGO 원칙을 이해하고 데이터의 품질(다양성, 정제)을 점검하는 역량.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "clean_data": "장난감 모형 사진은 빼야 한다.",
        "bias_opinion": "빨간 사과만 가르치면 초록 사과를 구별 못하기 때문"
      }
    }
    ```

### [D-3] 잘못 분류되는 경우 찾기
*   **연결 역량**
    *   **[K]**: **K4.1**: AI가 패턴 인식은 잘하나 맥락(Context) 이해 및 윤리 추론은 부족함을 인지
    *   **[S]**: **Critical Thinking**: 산출물의 오류와 정확성을 객관적인 기준으로 평가함
*   **측정 목표**: 기계 시각의 한계를 꼬집어내고 취약점을 경고하는 '모델 카드' 작성 능력 측정.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "analysis_text": "강아지의 숨결이나 움직임을 이해하지 못하고 픽셀 패턴만 봄",
        "model_card_limit": "경고: 갈색 둥근 빵 입력 시 오작동 확률 높음"
      }
    }
    ```

### [D-4] 학교 문제 돕는 AI 상상
*   **연결 역량**
    *   **[K]**: **K3.2**: 과업 중 반드시 인간의 개입(Intervention)이 필요한 지점 설계
    *   **[S]**: **Problem Solving**: 실생활 문제를 해결하기 위해 AI의 역량과 위험을 종합적으로 기획
    *   **[A]**: **Innovative**: 기술을 활용해 세상의 문제를 긍정적으로 해결하려는 도전 정신
*   **측정 목표**: 윤리성과 안전 장치(HITL)를 탑재한 사회 문제 해결용 AI 생태계 최종 설계.
*   **플랫폼 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "dataset_plan": "1년 치 잔반량 데이터",
        "hitl_step": "알레르기가 있는지는 영양사 선생님이 반드시 2차 확인한다."
      }
    }
    ```

## 🎯 [최종 완성본] 20개 문항 - 16개 미션 삼각검증(Triangulation) 구조

자가 보고식 설문 점수(20문항)와 플랫폼 수행 로그(JSON)를 대조 분석하여, 학생의 주관적 인식과 실제 수행 역량의 일치 여부를 아래 5대 역량군으로 분류하여 검증함.

### 1. AI 이해 (AI Understanding)
*   **[문항 1] AI 작동 원리 대략적 이해** 🔗 **[D-1] 분류 원리 / [D-3] 잘못된 분류**
*   **[문항 3] 인간 학습 모방 사실 인지** 🔗 **[D-2] AI가 배우는 자료 모으기**
*   **데이터 대조 전략:** 주관적 이해도가 높다고 답변한 학생이 실제 분류 모델의 자의성을 인지하고 오염 데이터를 정제하는 객관적 수치(정확도, 시간)가 높은지 분석.

### 2. AI 윤리 (AI Ethics)
*   **[문항 5] 개인정보/저작물 보호** 🔗 **[C-3] 홍보물 / [C-4] 제작 윤리**
*   **[문항 6] 올바르고 공정한 사용** 🔗 **[E-4] 불공평한 AI 편향성 성찰**
*   **[문항 13~14] 정보 옳고 그름 / 출처 의심** 🔗 **[E-3] 추천(필터버블) 성찰**
*   **데이터 대조 전략:** 윤리가 높은 학생일수록 출처 표기 준수율이 높고, 알고리즘의 편향을 더 심도 있게 서술(텍스트 분석)하는지 대조.

### 3. AI 참여 (AI Engagement)
*   **[문항 2] 생활 미치는 영향 설명** 🔗 **[E-1] 우리 주변의 AI 찾기**
*   **[문항 10~11] 동료/AI 협력 수행 의지** 🔗 **[M-2] 사람이 할 일 / AI가 도울 일**
*   **[문항 12] AI와의 명확한 소통 효능감** 🔗 **[M-3] 정확한 지시(프롬프트)**
*   **데이터 대조 전략:** 협력 및 소통 자신감이 실제 프롬프트 공식의 정교함과 과업 분배 시의 주도적 리더십(JSON 대화 로그)으로 나타나는지 분석.

### 4. AI 활용 (AI Application)
*   **[문항 7] 도구 적절 활용 가능성** 🔗 **[M-1] 사용 판단 딜레마**
*   **[문항 8] 글/이미지 생성 창작자 효능감** 🔗 **[C-1] 이어쓰기 / [C-2] 프롬프트 역설계**
*   **[문항 16] 목적별 정보 통제/수정** 🔗 **[M-3] 프롬프트 지시 최적화**
*   **[문항 17] 정보 선택 및 활용 능력** 🔗 **[C-3] 홍보물 만들기(윤색 단계)**
*   **데이터 대조 전략:** 활용 역량이 높다고 답변한 학생이 실제 AI 출력물을 맥락에 맞게 수정(Refinement)한 흔적과 본인 기여도 평가 수치 대조.

### 5. AI 안전 (AI Safety)
*   **[문항 4] 비정확성 유의 (환각 경계)** 🔗 **[E-2] AI 답 검증하기(채팅/논쟁)**
*   **[문항 9] 문제 해결 구상 능력** 🔗 **[D-4] 학교 AI 상상 설계(안전 장치 포함)**
*   **[문항 15] 문제 분해(Decompose) 능력** 🔗 **[M-2] 과업 구조화 로그**
*   **[문항 18~20] 자기효능감/끝없는 도전** 🔗 **[전 미션 이탈 및 재시도 로그]**
*   **데이터 대조 전략:** 안전 민감도가 높은 학생의 환각 반박 채팅 기록 및 설계 단계의 인간 개입(HITL) 반영 수준, 과업 분해의 논리성 분석.
