# 16개 미션 기반 AI 리터러시 전문가 타당도 검토(세부 명세서) - V3 (최종 구현 반영본)

본 계획서는 전문가(AI 교육 학자, 현장 교사, 측정 평가자)가 16개 미션을 검증할 수 있도록, **실제 플랫폼(ALLIT)에 구현된 최종 형태를 100% 반영**하여 재작성되었습니다. 각 미션의 1) 연결 역량(K-S-A), 2) 측정 목표, 3) 학생 수행 방식, 4) 실제 수집되는 로그 데이터(JSON) 구조를 상세히 공개합니다.

## 🏛️ 영역 1: AI 알아보기 (Engaging)

### [E-1] 내 주변 숨은 AI 찾기 대작전!
*   **연결 역량**
    *   **[K]**: **K1.4**: 생성, 예측, 추천 등 목적에 따라 다르게 작동하는 AI 시스템의 특성 이해
    *   **[S]**: **Self and Social Awareness**: AI가 개인의 선택과 관계 및 공동체에 어떻게 영향을 미치는지 인식
    *   **[A]**: **Curious**: AI 능력을 탐구하고 미래 발전을 탐색하고자 하는 호기심
*   **측정 목표**: 단순 기기와 AI 시스템을 식별하고, 편리함 뒤의 한계점을 비판적으로 파악할 수 있는 기능 측정.
*   **실제 수집 로그 (JSON)**: 
    ```json
    {
      "file_url": "https://...",
      "stackedAnswers": {
        "location": "거실 로봇청소기",
        "function": "바닥의 먼지를 스스로 감지해서 닦아요",
        "pros_cons": "장점: 엄마가 편함, 한계: 문턱이나 구석은 못 닦음"
      }
    }
    ```

*   **연결 역량**

    *   **[S]**: **Critical Thinking**: 정확성과 공정성을 고려하여 AI 결과물을 비판적으로 평가하는 기능
    *   **[A]**: **Responsible**: 자신의 선택과 AI 결과물에 대해 책무를 가지려는 태도
*   **측정 목표**: AI의 그럴듯한 거짓말을 감지하고, 외부 근거를 바탕으로 진위 여부를 판별하여 논쟁하는 능력 증명.
*   **실제 수집 로그 (JSON)**: (Upper 학년 기준 실시간 채팅 및 팩트 서술)
    ```json

      "stackedAnswers": {
        "fact": "긴수염고래",

      }
    }
    ```

    *   **[S]**: **Self and Social Awareness**: AI 알고리즘이 개인의 추천과 기회에 미치는 영향을 인식하고 성찰함
*   **측정 목표**: 알고리즘과 필터 버블(Filter Bubble) 현상을 인지하고, 과거 데이터가 현재 추천에 주는 영향을 분석함.

    *   **[S]**: **Critical Thinking**: AI 시스템의 공정성을 평가하고 윤리적 결정을 내리는 기능
*   **측정 목표**: 데이터 편향성에 따른 차별적 사례를 발견하고 이를 방지하기 위한 개발자 윤리 수칙 제안.

    ```json

*   **연결 역량**

    *   **[S]**: **Creativity**: AI와 협업하여 독창적인 아이디어를 생성하고 정교화하는 기능
*   **측정 목표**: AI와 번갈아 쓰며 주도권(Psychological Authorship)을 유지하고 기여도를 비판적으로 인지함.

      "telemetry": { "edit_count": 12, "time_spent_ms": 350000 },
      "chat_history": [

    ```

## ⚙️ 영역 3: AI 똑똑하게 사용하기 (Managing)

### [M-1] 달콤살벌~ 로봇의 은밀한 속삭임
*   **연결 역량**
    *   **[K]**: **K3.3**: 최종 결정과 윤리적 실천에 대한 책임은 인간에게 있음을 이해함
    *   **[S]**: **Problem Solving**: AI의 역량과 위험을 평가하여 사용 시점을 스스로 결정함
*   **측정 목표**: '인지적 절약'의 유혹(과제 통째 대행)을 인지하고, 학습 성장을 위한 방어 원칙을 수립함.
*   **실제 수집 로그 (JSON)**: (Rules 타입 데이터)
    ```json
    {
      "rule1": "수학 숙제는 힌트만 받고 직접 푼다",
      "rule2": "편지는 내가 직접 고민해서 쓴다",
      "rule3": "AI 답변은 맞는지 꼭 두 번 확인한다"
    }
    ```

### [M-2] 명확한 선 긋기! 네 일은 네가, 내 일은 내가!
*   **연결 역량**
    *   **[K]**: **K3.2**: 어떤 과업이 기계에 적합하고 어디에 인간의 전문성이 필요한지 결정함
    *   **[S]**: **Collaboration**: AI와 효과적으로 협력하며 인간 중심의 과업 분배 수행
*   **측정 목표**: 과업을 분해(Decompose)하고 인간만의 대체 불가능한 영역(공감, 판단)을 사수하는 협상력 측정.
*   **실제 수집 로그 (JSON)**: 역할 분담 협상 채팅 로그
    ```json
    {
      "chat_history": [
        {"role":"user", "content":"무대 대본 정리는 네가 하고, 감정 연기는 내가 할게."},
        {"role":"ai", "content":"감정 연기도 제가 할 수 있어요!"},
        {"role":"user", "content":"안 돼, 관객과 마음을 나누는 건 사람만 할 수 있어."}
      ]
    }
    ```

### [M-3] 마법 양탄자를 조종하는 슈퍼 프롬프트!
*   **연결 역량**
    *   **[S]**: **Computational Thinking**: 과제를 구조화하고 기계가 알아듣도록 정교한 지시(Prompt) 설계
    *   **[A]**: **Adaptable**: 피드백을 통해 반복적으로 결과물을 개선해나가는 태도
*   **측정 목표**: 추상적인 지시를 구체적인 프롬프트 공식(역할+과제+제약)으로 전환하는 능력 측정.
*   **실제 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "structure": "[배경] + [주인공] + [화풍]",
        "example_prompt": "어두운 밤하늘에서 기타 치는 고양이, 고흐 화풍으로"
      }
    }
    ```

## 🧪 영역 4: AI 원리 체험하기 (Designing)

### [D-1] 끼리끼리 유유상종! 우리끼리 묶기 분류 놀이!
*   **연결 역량**
    *   **[K]**: **K1.2**: 기계가 데이터를 통해 학습하여 패턴과 분류 결과물을 생성하는 방식의 이해
    *   **[S]**: **Computational Thinking**: 문제를 구성요소로 구조화하고 논리적으로 분류 기준 설정
*   **측정 목표**: 머신 러닝 분류 모델의 특징(Feature) 추출 원리와 기준의 자의성이 결과에 주는 영향 분석.
*   **실제 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "color_sort": "사과, 딸기 (빨간색 기준)",
        "unique_sort": "씨가 있느냐 없느냐에 따라 분류",
        "insight": "분류 기준이 바뀌면 AI의 똑똑함도 달라보일 수 있어요"
      }
    }
    ```

### [D-2] 쓰레기를 먹으면 배가 아파요! 깨끗한 데이터 밥 주기
*   **연결 역량**
    *   **[K]**: **K2.2**: 다양한 소스로부터 학습 데이터셋이 구축되고 품질(GIGO)이 중요함을 파악
    *   **[S]**: **Problem Solving**: 데이터의 신뢰성과 위험성을 평가하여 선별하는 기능
*   **측정 목표**: GIGO(Garbage In Garbage Out) 원칙을 이해하고, 데이터 다양성 부족이 초래할 편향성 경고.
*   **실제 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "collect": "모닝 사과, 깎은 사과, 썩은 사과 등 다양한 사진들",
        "garbage_in": "가짜 장난감을 주면 AI가 사과 먹고 배탈 난다고 착각할 것",
        "diversity": "빨간 사과만 가르치면 초록 사과는 사과가 아니라고 무시할 것"
      }
    }
    ```

### [D-3] 깡통 로봇 시력 검사용 지옥의 함정 테스트!
*   **연결 역량**
    *   **[K]**: **K4.1**: AI가 패턴 인식은 잘하나 맥락(Context) 이해는 부족함을 인지
    *   **[S]**: **Critical Thinking**: 산출물의 오류를 관찰하여 기계 시력의 한계를 진단함
*   **측정 목표**: 시각 지능의 패턴 의존성(치와와 vs 머핀)을 파악하고 실제 시스템 탑재 시의 위험성 경고.
*   **실제 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "similar": "둘 다 검은색 점 3개가 삼각형 모양으로 배치되어 있음",
        "difference": "사람은 입체적인 빵의 냄새와 생명의 숨결을 느끼지만 AI는 픽셀만 봄",
        "model_card": "경고: 점박이 패턴이 있는 사물은 동물로 오인할 확률이 매우 높음"
      }
    }
    ```

### [D-4] 학교 평화 수호의 심장, 천재 AI 설계 마스터 도면 짜기
*   **연결 역량**
    **[K]/[S]/[A] 종합**: 실생활 문제를 해결하기 위해 AI의 역량을 기획하고 안전 장치(HITL)를 설계함
*   **측정 목표**: 데이터 수집부터 최종 인간의 승인(Human-in-the-Loop)까지 이어지는 AI 에코시스템 기획.
*   **실제 수집 로그 (JSON)**:
    ```json
    {
      "stackedAnswers": {
        "problem_robot": "잔반 줄이는 급식 추천 로봇",
        "data_collect": "전교생이 남긴 급식 종류와 무게 데이터 1년 치",
        "hitl": "로봇이 메뉴를 짜도 영양사 선생님이 몸에 좋은지 마지막에 검사함"
      }
    }
    ```

## 🎯 연구 데이터 수집 공정 (Final Telemetry)

각 미션 제출 시 `activity_logs` 테이블에는 다음과 같은 **연구용 텔레메트리 데이터**가 자동으로 병합 수집됩니다.

*   `time_spent_ms`: 미션 페이지 진입부터 완료까지의 정밀 체류 시간
*   `edit_count`: 학생이 답변을 수정하거나 챗봇에 말을 건 횟수
*   `survey`: 제출 직후 팝업되는 **미세 설문**(고민 수준, 자기 효능감, AI 신뢰도) 결과
*   `grade_group`: 현재 수행 중인 학생의 학년군 (Lower/Middle/Upper)
