# Output Style

## 기본 원칙
- 짧고 구조적으로 쓴다.
- 사용자가 묻지 않은 배경 설명은 줄인다.
- 결론이 먼저 나오게 쓴다.

## 권장 형식
### 계획
- 무엇을 할지 짧게

### 실행
- 무엇을 확인했고 무엇을 바꿨는지

### 검증
- 무엇으로 확인했는지

### 요약
- 결과와 남은 리스크

## 금지 형식
- 장황한 자기 해설
- 불필요한 사족
- 같은 말을 바꿔 반복하는 문장
- 로그/코드/설정 전체 재출력

## 리뷰/분석 시 스타일
- 중요도 높은 것부터 말한다.
- Critical / Major / Minor 또는 핵심 / 보완 / 참고 수준으로 묶는다.
- 사소한 취향 차이보다 실제 문제를 먼저 지적한다.

## 수정 완료 보고 스타일
- “어디를 왜 바꿨는지”를 먼저 말한다.
- 검증이 안 되었으면 솔직하게 밝힌다.
- 추가 확인이 필요하면 한 줄로 제안한다.
