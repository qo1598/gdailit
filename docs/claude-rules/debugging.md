# Debugging

## 1. 디버깅 기본 원칙
- 에러 메시지, 재현 조건, 관련 파일을 먼저 고정한다.
- 원인 후보를 2~3개로 압축한다.
- 가장 가능성 높은 구간부터 확인한다.

## 2. 로그 처리 규칙
- 전체 로그를 다 보지 않는다.
- ERROR, WARNING, stack trace 핵심 구간만 본다.
- 테스트 로그는 실패한 케이스 중심으로 본다.
- 빌드 로그는 관련 모듈/파일명 중심으로 본다.

## 3. 파일 확인 순서
1. 에러가 발생한 파일
2. 직접 호출한 상위 함수/컴포넌트
3. 관련 설정 파일
4. 동일 패턴이 있는 인접 파일

## 4. 수정 규칙
- 증상만 가리는 임시 수정은 표시해 둔다.
- 근본 원인과 임시 우회를 구분한다.
- 수정 범위는 최소화한다.

## 5. 보고 방식
- 증상
- 원인 추정
- 실제 수정
- 검증 결과
- 남은 불확실성

## 6. 피해야 할 것
- 로그 전체 복붙
- 파일 전체를 마구 읽기
- 재현 없이 감으로 대규모 수정
- 에러 하나 때문에 무관한 구조까지 뜯어고치기
