# M영역 평가 루브릭 v1 — 연구자 코딩 가이드

_기반 데이터: V4/M-1.js ~ M-4.js · 2026-04-27_

---

## 1. 설계 원칙

| 항목 | 내용 |
|---|---|
| **목적** | 연구자가 학생 응답 JSON을 직접 읽고 AI 관리·위임 역량 수준을 수동 코딩, 정량 분석에 활용 |
| **코딩 단위** | 미션 카드(M-1-L ~ M-4-H) × 준거 |
| **평가 척도** | 준거별 **0·1·2** (3점 척도) |
| **최종 수준** | 준거 합산 점수 → **0·1·2·3** (4수준) 변환 |
| **적용 시점** | 학생 수행 완료 후, `mission_submissions_v4.answers` JSONB + `ai_interactions` 테이블 사후 코딩 |
| **신뢰도 목표** | 전체 표본의 20% 이상 두 코더 독립 채점 → Cohen's κ ≥ 0.70 |

### E·C 영역과의 차이점

| 구분 | E영역 | C영역 | **M영역** |
|---|---|---|---|
| **영역 특성** | AI 출력물 관찰·평가 | AI와 협업 창작 | **AI에 위임·감독·거버넌스** |
| **공통 축** | 인식/근거/판단/제안 | 의도/협업/정제/기여/공유 | **위임/실행/위험/전략/성찰** |
| **핵심 판단** | "이게 맞나?" | "어떻게 만들까?" | **"이걸 맡길까 말까?"** |
| **산출물 성격** | 판단 결과 | 창작물 | **전략표·규칙·가이드·매뉴얼** |
| **AI 상호작용** | 없음 | 실시간 창작 | **위임 실험·비교·검증** |

## 2. JSON 접근 규칙

E/C 영역과 동일한 uiMode에 더해, M영역에서 새로 등장하는 패턴:

| uiMode | `stepN` 값 shape | M영역 주요 사용 |
|---|---|---|
| `classify_cards_carousel` | `{[cardId]: groupId}` | M-1-L 숙제 분류, M-2-L 정보 분류 |
| `ai_chat_turn` | `{input, aiResponse, retryCount}` | 전 미션 (위임 실험) |
| `per_case_judge` | `{[caseId]: {judgment, reasons?, text?}}` | 비교 평가, 상황 판단 |
| `multi_select_chips` | `string[]` | 문제 발견, 약속 선택 |
| `multi_free_text` | `{[questionId]: string}` | 전략표, 규칙, 가이드라인 |
| `free_text` | `string` | 성찰, 최종 판단 |
| `single_select_cards` | `string` | 실험 과제 선택, 판단 |

### `ai_interactions` 테이블 참조 (M영역 전 미션 해당)

M영역은 **모든 미션에 AI 상호작용**이 포함되므로, `ai_interactions`를 반드시 참조:

| 필드 | M영역 코딩 활용 |
|---|---|
| `attempt` | 위임 시 재시도 횟수 (집요함/포기 패턴) |
| `user_prompt` | 위임 지시의 구체성 수준 판단 |
| `student_action` | 결과 수용·기각 판단 |
| `fallback_used` | API 실패 시에도 실험을 이어갔는가 |

## 3. 공통 평가 축 (M영역)

M영역은 OECD AILit "Managing AI" 정의에 기반한 5개 평가 축을 사용한다.

| 축 | 이름 | 핵심 질문 | 측정 대상 |
|---|---|---|---|
| **A** | **위임 판단** | AI에게 맡길 일과 직접 할 일을 적절히 구분했는가? | 과제 분류 정확성, 위임 적합성 판단 |
| **B** | **실행과 감독** | AI에게 실제로 맡기고 결과를 확인·검증했는가? | 실험 수행, 결과 점검, 비교 분석 |
| **C** | **위험 관리** | AI 사용의 위험(오류·정보·의존)을 인식하고 대비했는가? | 위험 식별, 안전 전략, 대응 방안 |
| **D** | **전략 수립** | 경험에 기반한 실행 가능한 전략·규칙을 만들었는가? | 위임 전략, 사용 가이드, 거버넌스 규정 |
| **E** | **판단 성찰** | AI와 인간의 역할 차이를 이해하고 자기 판단을 설명할 수 있는가? | 메타인지, 책임 인식, 근거 기반 설명 |

### 축 적용 원칙

- 학년 올라갈수록 축 수 증가: L은 A+B+E (3축), M은 A+B+C+D (4축), H는 A+B+C+D+E (5축)
- M-2·M-4는 C축(위험 관리) 중심, M-1·M-3은 A축(위임 판단)+D축(전략) 중심

## 4. 공통 코딩 척도

| 점수 | 명칭 | 기준 |
|---|---|---|
| **0** | 없음 | 응답 없음 / null / 무작위 / 과제와 무관한 내용 |
| **1** | 부분 | 해당 역량의 흔적이 보이나 불완전하거나 표면적 수준 |
| **2** | 명확 | 준거 충족 지표를 충분히 달성 |

> **객관형 준거**: JSON 값·개수·패턴으로 기계적 확인
> **주관형 준거** (free text / multi_free_text): 명시된 핵심어·개념 존재 여부로 확인
> **AI 상호작용 준거**: `ai_interactions` 테이블의 attempt·user_prompt·student_action 참조
> **이진 준거** (⬩): 0 또는 2만 사용, 1 미사용

## 8. 코딩 절차 (E·C 영역과 동일)

1. `mission_submissions_v4` 테이블에서 해당 학생·카드 row의 `answers` JSONB 열람
2. `ai_interactions` 테이블에서 같은 `session_id`의 AI 호출 기록 열람 (M영역 전 미션 해당)

4. JSON 필드 경로로 값 확인 → 0/1/2 기록
5. 합산 → 변환표 적용 → 최종 수준 기록
6. inter-rater 20% 표본 독립 채점 → κ ≥ 0.70
