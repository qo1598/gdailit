# 💎 16개 미션 초정밀 기술 설계도 (The Ultimate Blueprint)

이 문서는 16개 모든 미션에 대해 학년군별 **구체적 발문(Prompt)**, **수행 과제(Action)**, 그리고 **데이터 저장 형식(Full JSON)**을 단 하나의 예외 없이 상세히 정의합니다.

---

## 🏛️ 영역 1: AI 알아보기 (Engaging)

### [E-1] 우리 주변의 AI 찾기
- **역량 코드**: K1.4 (AI 식별), S1.1 (서비스 탐색)
- **학년별 설계**:
    - **[저] (1-2)**: 
        - **발문**: "우리 집이나 학교에서 AI가 들어있는 기계나 서비스를 찾아 사진을 찍어볼까요?"
        - **과제**: 사진 업로드 + "어디서 찾았나요?" 단문 입력
    - **[중] (3-4)**: 
        - **발문**: "+ 이 AI는 우리에게 어떤 도움을 주나요? AI가 하는 일을 설명해 보세요."
        - **과제**: 누적 입력 (사진 + 장소 + 기능 설명)
    - **[고] (5-6)**: 
        - **발문**: "+ 이 AI 덕분에 편리한 점은 무엇이고, 혹시 아쉬운 점이나 한계는 무엇인가요?"
        - **과제**: 누적 입력 (사진 + 장소 + 기능 + 장단점 분석)
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "E-1",
  "data": {
    "photo_url": "url",
    "location": "거실",
    "function": "먼지 청소",
    "analysis": { "benefit": "시간 절약", "limit": "전선 꼬임" }
  }
}
```

### [E-2] AI 답 검증하기
- **역량 코드**: K4.1, S3.2 (비판적 사고)
- **학년별 설계**:
    - **[저/중] (1-4) [AI 선제시형]**:
        - **AI 발문**: "지구에서 두 번째로 큰 동물은 호랑이예요!"
        - **학생 발문**: "AI의 말이 진짜일까요? 정답인지 아닌지 선택하고 이유를 써보세요."
        - **과제**: OX 선택 + 이유 텍스트 입력
    - **[고] (5-6) [실시간 반박형]**:
        - **학생 발문**: "AI에게 어려운 질문을 던져보고, 틀린 답을 하면 다른 자료를 근거로 AI와 논쟁해 보세요."
        - **과제**: 실시간 채팅 (3턴 이상)
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "E-2",
  "data": {
    "judgment": "false",
    "reason": "호랑이는 바다 생물이 아님",
    "chat_history": [
      {"role": "user", "content": "..."},
      {"role": "ai", "content": "..."}
    ]
  }
}
```

### [E-3] 추천은 왜 나에게 왔을까?
- **역량 코드**: K1.1, K4.3 (알고리즘)
- **학년별 설계**:
    - **[저] (1-2)**: 
        - **발문**: "유튜브나 장난감 가게에서 나에게 추천해준 영상이나 물건은 무엇인가요?"
        - **과제**: 추천받은 항목 이름 입력
    - **[중] (3-4)**: 
        - **발문**: "+ AI는 왜 나에게 그것을 추천했을까요? 내가 예전에 무엇을 했는지 떠올려보세요."
        - **과제**: 추천 원인 추측 서술
    - **[고] (5-6)**: 
        - **발문**: "+ 계속 같은 것만 추천받으면 어떤 문제가 생길까요? 나의 취향이 좁아지지는 않을까요?"
        - **과제**: 필터 버블 및 알고리즘 편향성 성찰 서술
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "E-3",
  "data": {
    "rec_item": "슬라임 영상",
    "guess_reason": "어제 슬라임 검색함",
    "critical_reflection": "다른 영상을 보기 힘들어지는 편식이 생길 수 있음"
  }
}
```

### [E-4] AI는 누구에게 불공평할까?
- **역량 코드**: K2.1, K2.5 (편향성 및 윤리)
- **학년별 설계**:
    - **[저] (1-2)**: 
        - **발문**: "누구에게나 공평한 AI가 되려면 모든 사람의 사진을 골고루 공부해야 할까요?"
        - **과제**: OX 퀴즈 응답
    - **[중] (3-4)**: 
        - **발문**: "+ AI가 특정 사람(피부색, 나이 등)을 잘 알아보지 못하는 사례가 있다면 찾아보세요."
        - **과제**: 불공평한 구체 사례 서술
    - **[고] (5-6)**: 
        - **발문**: "+ 이런 불공평함을 고치기 위해 AI 개발자가 지켜야 할 '윤리 수칙' 3개를 제안하세요."
        - **과제**: 3가지 윤리 원칙 명문화
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "E-4",
  "data": {
    "fairness_check": true,
    "bias_case": "특정 피부색 인식 불가 사례",
    "ethics_policy": ["다양한 데이터 사용", "차별성 검사 필수", "사람의 최종 확인"]
  }
}
```

## 🎨 영역 2: AI와 함께 만들기 (Creating)

### [C-1] AI와 이야기 이어쓰기
- **역량 코드**: K4.1, S5.2 (협업 창작)
- **학년별 설계**:
    - **[저/중] (1-4) [AI 선제시형]**:
        - **AI 발문**: "옛날 옛적 초콜릿 숲에 투명 망토를 쓴 토끼가 살았어요..."
        - **학생 발문**: "AI가 시작한 이야기에 딱 한 문장만 멋지게 덧붙여서 완성해 볼까요?"
    - **[고] (5-6) [양방향 대화형]**:
        - **학생 발문**: "AI와 대화하며 한 편의 동화를 완성하세요. 다 쓴 뒤 내가 기여한 비중을 %로 나타내보세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "C-1",
  "data": {
    "story_final": "토끼가 초콜릿 집을 발견했어요.",
    "chat_log": [...],
    "meta": { "user_contribution_pct": 40, "ai_help_point": "배경 묘사" }
  }
}
```

### [C-2] AI 그림 보고 새 장면 만들기
- **역량 코드**: K1.4 (생성 원리)
- **학년별 설계**:
    - **[저] (1-2)**: "AI 화가가 그린 신기한 그림 3장 중 가장 마음에 드는 것을 골라주세요." (이미지 선택)
    - **[중] (3-4)**: "+ 이 그림에서 AI가 실수로 잘못 그린 부분이 있나요? 왜 그렇게 생각하는지 써보세요."
    - **[고] (5-6)**: "+ 이 그림을 AI가 만들 때 어떤 전문적인 명령어(프롬프트)를 썼을지 추측해 보세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "C-2",
  "data": {
    "selected_img": "id_02",
    "error_analysis": "나뭇잎 모양이 이상함",
    "prompt_reverse_engineering": "Impressionism style, sunlight through leaves"
  }
}
```

### [C-3] AI와 홍보물 만들기
- **역량 코드**: K2.3 (결과물 정교화)
- **[저] (1-2)**: "축하 파티 소식! AI가 추천한 문구 3개 중 가장 마음에 드는 것을 고르세요."
- **[중] (3-4)**: "+ 고른 문구를 우리 학교나 우리 반의 이름이 들어가게 더 멋지게 고쳐보세요."
- **[고] (5-6)**: "+ 이 홍보물을 만들 때 다른 사람의 그림이나 글을 베끼지 않았는지 최종 점검표를 만드세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "C-3",
  "data": {
    "original_pick": "오늘의 주인공은 너!",
    "refined_text": "우리 경동초 1반의 주인공은 바로 너!",
    "integrity_check": ["이미지 저작권 확인", "출처 표시 완"]
  }
}
```

### [C-4] 누가 만들었을까?
- **역량 코드**: K5.3 (진위 및 지식재산)
- **[저] (1-2)**: "사람이 그린 그림과 AI가 그린 그림이 섞여 있어요. 구별할 수 있을까요? (퀴즈)"
- **[중] (3-4)**: "+ 내가 만약 AI의 도움을 받았다면, 사람들에게 어떻게 알려야 정직하고 매너 있는 행동일까요?"
- **[고] (5-6)**: "+ 앞으로 내가 AI를 활용해 창작할 때 지켜야 할 '나만의 정직 약속'을 한 문장으로 쓰세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "C-4",
  "data": {
    "quiz_result": 80,
    "attribution_opinion": "AI가 만든 부분에 표시를 남긴다",
    "my_ethics_code": "AI의 도전을 나의 노력으로 속이지 않는다"
  }
}
```

## ⚙️ 영역 3: AI 똑똑하게 쓰기 (Managing)

### [M-1] 이럴 때 AI 써도 될까?
- **역량 코드**: K4.1, K5.4 (윤리적 판단)
- **[저] (1-2)**: "숙제를 하기 싫을 때 AI가 대신 다 해주겠다고 하면? (상황극 리액션 선택)"
- **[중] (3-4)**: "+ AI에게 숙제를 맡기는 것과 내가 직접 고민하는 것 중 무엇이 더 성장에 도움이 될까요?"
- **[고] (5-6)**: "+ 공부할 때 AI를 '커닝 도구'가 아닌 '학습 도우미'로 쓰기 위한 나만의 기준 3가지를 만드세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "M-1",
  "data": {
    "reaction": "reject",
    "value_judgment": "직접 해야 내 실력이 됨",
    "usage_rules": ["답만 베끼지 않기", "모르는 개념 질문하기", "힌트만 얻기"]
  }
}
```

### [M-2] 사람이 할 일, AI가 도울 일
- **역량 코드**: K4.1 (문제 분해)
- **[저/중] (1-4) [AI 제안형]**:
    - **AI 발문**: "학교 축제 준비! 저는 포스터를 디자인할게요. 학생은 친구들을 초대하는 초대장을 써줄래요?"
    - **학생 발문**: "AI의 제안이 마음에 드나요? 역할이 적절한지 판단해 보세요."
- **[고] (5-6) [비판적 협상형]**:
    - **학생 발문**: "AI와 업무를 나눌 때의 '나만의 원칙'을 먼저 정하고, 그 원칙에 따라 AI와 역할을 협상하여 분담표를 만드세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "M-2",
  "data": {
    "accept_ai_proposal": true,
    "negotiation_log": [...],
    "final_roles": { "human": "기획", "ai": "실행 보조" }
  }
}
```

### [M-3] AI에게 정확히 부탁하기
- **역량 코드**: K1.3, K2.3 (프롬프팅)
- **[저] (1-2)**: "AI에게 '우주선 그려줘'라고 말하고 결과를 확인하세요."
- **[중] (3-4)**: "+ 이번에는 '무지개 빛깔 꼬리가 달린 초록색 우주선'처럼 더 자세하게 부탁해 보세요."
- **[고] (5-6)**: "+ AI가 내 마음을 찰떡같이 알아듣고 결과를 내놓게 만드는 '마법의 지시문 공식'을 만드세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "M-3",
  "data": {
    "try_1": "우주선",
    "try_2": "초록색 우주선",
    "prompt_formula": "주제 + 색깔 + 배경 + 느낌"
  }
}
```

### [M-4] 우리 반 AI 사용 약속
- **역량 코드**: K5.4 (거버넌스)
- **[저] (1-2)**: "교실에서 AI를 쓸 때 지켜야 할 질서가 무엇인지 옳은 행동 카드를 골라보세요."
- **[중] (3-4)**: "+ 우리 반 친구들이 모두 기분 좋게 AI를 쓰기 위해 꼭 필요한 약속 3가지를 제안하세요."
- **[고] (5-6)**: "+ 친구들의 의견을 모아 완성한 '우리 반 AI 사용 공동 선언문'을 멋진 문장으로 완성하세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "M-4",
  "data": {
    "selected_good_behaviors": [1, 3],
    "proposals": ["서로 돕기", "정직하기", "존중하기"],
    "class_oath": "우리는 AI를 도구로 삼아 더 넓은 세상을 탐구하겠습니다."
  }
}
```

## 🧪 영역 4: AI 원리 체험하기 (Designing)

### [D-1] 같은 것끼리 묶어보자
- **역량 코드**: K1.2 (분류 모델)
- **[저] (1-2)**: "여러 가지 블록 사진이 있어요. 모양이 같은 것끼리 드래그해서 묶어볼까요? (혹은 선택)"
- **[중] (3-4)**: "+ 모양 말고 나만의 새로운 기준으로 블록을 분류해 보세요. 그 기준의 이름은 무엇인가요?"
- **[고] (5-6)**: "+ 분류 기준이 달라지면 AI의 판단 결과는 어떻게 바뀔까요? 기준 설정의 중요성을 서술하세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "D-1",
  "data": {
    "basic_cluster": "by_shape",
    "custom_cluster": "by_material",
    "analysis": "데이터의 특징을 어떻게 정하느냐가 AI의 성능을 결정함"
  }
}
```

### [D-2] AI가 배우는 자료 모으기
- **역량 코드**: K1.2, K2.2 (데이터 리터러시)
- **[저] (1-2)**: "AI가 '사과'가 무엇인지 배우려면 어떤 사진들이 필요할까요? 사과 사진 3장을 찾아 올리세요."
- **[중] (3-4)**: "+ 모은 사진 중에 썩은 사과나 그려진 사과가 섞여 있다면 AI가 헷갈려 할까요? 좋은 데이터를 골라보세요."
- **[고] (5-6)**: "+ 만약 장미 사진만 보여주고 '꽃'을 배우게 하면 어떤 문제가 생길까요? 데이터 다양성의 필요성을 쓰세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "D-2",
  "data": {
    "dataset": ["url1", "url2", "url3"],
    "clean_data_check": ["bad_data_removed"],
    "bias_opinion": "편향된 데이터는 AI의 시야를 좁게 만듦"
  }
}
```

### [D-3] 잘못 분류되는 경우 찾기
- **역량 코드**: K1.2, K2.1 (모델 한계)
- **[저] (1-2)**: "AI가 가끔 헷갈려 하는 사물 사진(예: 머핀 vs 강아지)을 보며 어디가 닮았는지 찾아보세요."
- **[중] (3-4)**: "+ AI가 왜 이 사진들을 헷갈려 할까요? AI의 '눈' 입장에서 시각적 특징을 분석해 보세요."
- **[고] (5-6)**: "+ 이 AI 모델의 특징과 한계를 요약한 **'모델 카드(Model Card)'**의 빈칸을 채워 완성해 보세요."
- **데이터 저장 예시 (JSON)**:
```json
{
  "mission_id": "D-3",
  "data": {
    "visual_similarity": "색감과 질감이 유사함",
    "model_card": { "target": "개 인식기", "limit": "배경이 복잡하면 오인식률 증가" }
  }
}
```

### [D-4] 학교 문제 돕는 AI 상상
