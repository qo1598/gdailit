# 🌀 16개 미션 누적 나선형 구조 상세 설계 (Detailed Spiral Map)

이 설계도는 `mission_implications_tablev2.md`를 기반으로, 각 미션이 학년군별로 어떻게 **누적(Cumulative)**되며 심화되는지 보여줍니다. 상위 학년은 하위 학년의 활동을 포함하면서 새로운 과제를 추가로 수행합니다.

---

## 🤖 AI 상호작용형 미션 (Interactive Missions)

16개 미션 중 **채팅(Chat) 형태**로 AI와 직접 대화하며 해결하는 미션은 다음과 같습니다.

### 1. **C-1: AI와 이야기 이어쓰기 (AI Storytelling Assist)**
- **[저/중] (1-4) - AI 선제시형**: 
    - 학생이 시작 버튼을 누르면 AI가 재미있는 이야기의 첫 단락을 먼저 생성하여 보여줍니다. 
    - 학생은 AI가 만든 내용을 읽고 그 뒤에 이어질 한 문장을 덧붙이거나 간단한 수정 의견을 남깁니다. (비계 설정 강조)
- **[고] (5-6) - 양방향 대화형**: 
    - 학생과 AI가 번갈아 가며 실시간 채팅으로 이야기를 교환합니다. 
    - AI의 제안을 비평하거나 대화 중간에 이야기의 방향을 완전히 트는 고차원적 협업을 수행합니다.

### 2. **M-2: 사람이 할 일, AI가 도와줄 일 (Role Division Negotiation)**
- **[저/중] (1-4) - AI 제안형**: 
    - 상황(예: 학교 급식 시간)을 정하면 AI가 "제가 이런 일들을 도와드릴 수 있어요!"라며 역할 리스트를 먼저 제안합니다. 
    - 학생은 리스트를 보고 찬성/반대를 표시하거나 간단한 이유를 덧붙입니다.
- **[고] (5-6) - 비판적 협상형**: 
    - 학생이 먼저 상황과 필요 역할을 제시하고, AI와 각 역할의 효율성과 윤리적 측면에 대해 토론하며 최종 '역할 분담표'를 확정합니다.

### 3. **E-2: AI 답 그대로 믿어도 될까? (Fact-Checking Dispute)**
- **[저/중] (1-4) - 오류 탐색형**: 
    - AI가 특정 주제에 대해 3가지 정보를 제시합니다. (그중 하나는 틀린 정보) 
    - 학생은 AI의 답변 중 수상한 것을 골라내고 확인 버튼을 누르는 방식으로 참여합니다.
- **[고] (5-6) - 실시간 반박형**: 
    - 학생이 직접 질문을 던지고, AI의 답변에 대해 외부 자료를 근거로 실시간으로 반박하거나 오류 수정을 지시합니다.

### 💡 상호작용 구현 포인트
- **Persona Setting**: 각 미션마다 AI에게 다른 '역할(Role)'을 부여합니다 (예: 동화 작가, 역할 분담 전문가, 프롬프트 코치).
- **Turn Count**: 학년이 올라갈수록 미션 완료를 위한 **'최소 대화 턴 수'**를 늘려 인지적 노력을 유도합니다 (연구 가설 4 반영).
- **Log Collection**: 학생과 AI의 모든 대화 로그(Chat History)를 JSON 형태로 저장하여 '상호작용의 질'을 분석할 수 있게 합니다.

### 💡 구현 포인트
1. **입력 폼의 확장**: 고학년 페이지에서는 저학년용 입력창(예: 사진 업로드) 아래에 중학년용(설명), 고학년용(분석) 입력창이 **세로로 스택**되어 나타납니다.
2. **상태 관리**: `formData`가 단순 문자열이 아닌 `{ step1: '', step2: '', step3: '' }`와 같은 객체 구조로 확장되어야 합니다.
3. **가이드 텍스트**: 각 입력창마다 학년 수준에 맞는 구체적인 팁(Tip)을 제공하여 비계 설정(Scaffolding)을 지원합니다.
