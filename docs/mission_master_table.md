# 📑 AI 리터러시 플랫폼: 미션 총괄표 (Mission Master Table)

이 표는 16개 미션 각각에 대해 학년군별로 수행해야 할 **누적형(Cumulative)** 과제를 정의합니다. 모든 상위 학년은 하위 학년의 과제를 기본으로 포함하며, 새로운 심화 단계가 추가됩니다.

## 🏛️ 영역 1: AI 알아보기 (Engaging)

| 코드 | 미션 명칭 | [저] 1~2학년 (인식/경험) | [중] 3~4학년 (설명/적용) | [고] 5~6학년 (판단/성찰) | 상호작용 방식 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **E-1** | 우리 주변의 AI 찾기 | AI 기기/서비스 사진 업로드 | + 하는 일(기능) 설명 적기 | + 도움과 기술적 한계 분석 | 정적 입력 |
| **E-2** | AI 답 검증하기 | AI가 생성한 오답 탐색 (OX) | + 정보 교차 검증 이유 서술 | + AI와 실시간 답변 논쟁(Dispute) | **AI 선제시 → 실시간 채팅** |
| **E-3** | 추천 이유 추측하기 | 내가 받은 추천 사례 공유 | + 추천 알고리즘의 이유 추측 | + 알고리즘 편향성/환경 성찰 | 정적 입력 |
| **E-4** | 불공평한 AI 찾기 | 공평함에 대한 의견 (OX) | + 불공평한 구체적 사례 찾기 | + 기술적/윤리적 개선 방안 제안 | 정적 입력 |

## 🎨 영역 2: AI와 함께 만들기 (Creating)

| 코드 | 미션 명칭 | [저] 1~2학년 (인식/경험) | [중] 3~4학년 (설명/적용) | [고] 5~6학년 (판단/성찰) | 상호작용 방식 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **C-1** | AI와 이야기 이어쓰기 | AI가 선제시한 글에 한 문장 잇기 | + AI와 대화하며 내용 수정 | + AI 기여도 분석 및 오프로딩 성찰 | **AI 선제시 → 실시간 채팅** |
| **C-2** | AI 그림 장면 만들기 | 마음에 드는 AI 그림 선택 | + 그림의 특징/이상한 점 설명 | + 기술적 용어로 작업 방식 기술 | 정적 입력 |
| **C-3** | AI와 홍보물 만들기 | AI가 추천한 문구 고르기 | + 문구를 직접 수정/확장 | + 저작권 및 생성물 표시 최종 점검 | **AI 선제시 → 결과 수정** |
| **C-4** | 누가 만들었을까? | 사람 vs AI 작품 구분하기 | + AI 도움 표시(워터마크) 제안 | + 나만의 디지털 창작 윤리 수립 | 정적 입력 |

## ⚙️ 영역 4: AI 똑똑하게 쓰기 (Managing)

| 코드 | 미션 명칭 | [저] 1~2학년 (인식/경험) | [중] 3~4학년 (설명/적용) | [고] 5~6학년 (판단/성찰) | 상호작용 방식 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **M-1** | AI 사용 판단 기준 | 사용 가능 상황 선택 (OX) | + AI와 '효율 vs 윤리' 토론 | + 나만의 판단 기준 3가지 명문화 | **상황 상황극 → 실시간 채팅** |
| **M-2** | 사람이 할 일, AI 할 일 | AI가 제시한 역할안에 반응 | + 역할 배분의 구체적 이유 토론 | + AI와 협상을 통한 분담표 완성 | **AI 선제시 → 실시간 채팅** |
| **M-3** | AI에게 정확히 부탁하기 | 간단한 프롬프트 입력 및 결과 | + 구체적 조건(프롬프트) 덧씌우기 | + AI에게 고정 규칙(Persona) 부여 | **결과물 반복 수정(Iterative)** |
| **M-4** | 우리 반 AI 사용 약속 | 좋은/나쁜 사용 사례 선택 | + 우리 반 약속 문장 직접 작성 | + 공동체 선언문 및 서약 완성 | 정적 입력 |

## 🧪 영역 4: AI 원리 체험하기 (Designing)

| 코드 | 미션 명칭 | [저] 1~2학년 (인식/경험) | [중] 3~4학년 (설명/적용) | [고] 5~6학년 (판단/성찰) | 상호작용 방식 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **D-1** | 같은 것끼리 묶어보자 | 주어진 기준(색/모양) 분류 | + 나만의 분류 기준 제안 | + 기준 변경에 따른 결과값 비교 | 정적 입력 |
| **D-2** | AI가 배우는 데이터 | 관련 데이터(이미지 등) 수집 | + 데이터의 품질 선벌 (Good/Bad) | + 데이터 편향성 보완책 제안 | 정적 입력 |
| **D-3** | 잘못 분류되는 경우 | AI의 분류 오류 찾아내기 | + 오류 발생의 원인 추측/분석 | + **모델 카드(Model Card)** 작성 | 정적 입력 |
| **D-4** | 학교 문제 돕는 AI 상상 | 해결하고 싶은 문제 정의 | + 필요한 데이터셋 상세 기획 | + **인간 개입(HITL)** 설계 루틴 추가 | 정적 입력 |

---

### 📝 구현 핵심 전략
1.  **누적적 UI (Stacked UI)**: 고학년으로 갈수록 하위 학년의 입력폼이 아래에 누적되어 나타남 (이전 단계를 거쳐야 다음 단계 분석이 가능하도록 설계).
2.  **학년별 비계 설정 (Scaffolding)**:
    -   1~4학년: AI가 **먼저 자료를 제시**하고 학생이 반응하게 함 (부담 완화).
    -   5~6학년: 학생이 **먼저 대화를 시작**하고 AI와 논쟁/협상하게 함 (주도성 강화).
3.  **데이터 무결성**: 모든 단계의 입력값은 하나의 JSON 객체로 묶여 Supabase에 저장되어, 학년별 발달 수준 분석이 가능하도록 함.
